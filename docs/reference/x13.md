# X-13ARIMA-SEATS (x13)

The `tse.x13` subpackage is a MATLAB port of the `X13` module of
`TimeSeriesEcon.jl`. It exposes the X-13ARIMA-SEATS specification interface as
MATLAB objects, serialises a spec to the `.spc` text format (respecting X13's
strict line-length limit), runs the `x13as` binary, and returns the results as
`TSeries`/`MVTSeries` and structs.

!!! info "The binary is not bundled"
    Unlike the Julia package (which ships `X13as_jll`), the MATLAB port does
    **not** include the X-13ARIMA-SEATS executable. Install it from the US
    Census Bureau and point the package at it:

    ```matlab
    tse.setoption('x13path', '/path/to/x13as');
    ```

    Building and serialising specs (`tse.x13.x13write(spec, 'test', true)`) works
    without the binary; only `tse.x13.run` / `tse.x13.deseasonalize` need it.

## Building a spec

```matlab
import tse.*
ts  = TSeries(qq(2020,1), randn(40,1)*5 + 100);
xts = x13.series(ts, 'title', "Quarterly GDP");
spec = x13.newspec(xts);
x13.arima(spec, x13.ArimaModel(0,1,1,0,1,1));   % sets spec.arima in place
x13.x11(spec, 'mode', 'mult');
disp(x13.x13write(spec, 'test', true))           % the .spc text
```

Every spec constructor (`series`, `arima`, `automdl`, `pickmdl`, `regression`,
`x11`, `x11regression`, `transform`, `seats`, `estimate`, `check`, `forecast`,
`force`, `history`, `identify`, `metadata`, `outlier`, `slidingspans`,
`spectrum`) can be called two ways:

- `obj = tse.x13.NAME(...)` — returns the subspec object, or
- `tse.x13.NAME(spec, ...)` — sets `spec.NAME` in place (the Julia `NAME!` form;
  `X13spec` is a handle object).

## ARIMA models

`tse.x13.ArimaSpec(p, d, q[, period])` is one model factor;
`tse.x13.ArimaModel(...)` collects factors. Orders are scalars; an operator with
explicit (missing) lags is given as a cell, e.g. `ArimaSpec({2}, 1, 0)` writes
`([2] 1 0)` (a bare `[2]` and `2` are identical in MATLAB, so the cell is needed).
`ArimaSpec(p,d,q,P,D,Q)` returns a 1×2 array of two factors.

## Regression variables

The `variables` argument of `regression`/`x11regression` takes a cell mixing
char symbols and variable objects:

| Group | Constructors |
|-------|--------------|
| Single-date outliers | `ao ls tc so` |
| Date-range regressors | `aos lss rp qd qi tl` |
| Holiday / count | `easter labor thank sceaster easterstock tdstock tdstock1coef sincos` |
| Regime-change | `td tdnolpyear td1coef td1nolpyear lpyear lom loq seasonal` |

```matlab
x13.regression(spec, 'variables', {'const', x13.ao(qq(2008,3)), x13.td()});
```

`tse.x13.M(n)`, `Q(n)`, `H(n)` build fuzzy period markers and `tse.x13.Span`
builds a (possibly open-ended) span, e.g. `x13.Span([], x13.M(12))` → `(, 0.dec)`.

## Running

```matlab
tse.setoption('x13path', '/path/to/x13as');
res = x13.run(spec);          % a tse.x13.X13result
sa  = res.series.d11;         % final seasonally adjusted series (x11)
x13.descriptions(res)         % which outputs are available

sa2 = x13.deseasonalize(ts);  % convenience: default x11, returns d11
```

`X13result` holds `series`, `tables`, `text` and `other` structs plus parsed
`errors`/`warnings`/`notes`. `tse.x13.cleanup()` removes leftover temporary run
folders.

!!! info "Julia ↔ MATLAB"
    Julia `Symbol`s map to MATLAB **char** (`'mult'`) and Julia `String`s to
    double-quoted **string** (`"My title"`). The `name!`/`series!` mutating
    functions become `tse.x13.name(spec, ...)`. Not ported: the per-table English
    description text (`_output_descriptions`) and the bundled binary.
